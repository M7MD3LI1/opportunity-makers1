# توثيق مشروع "صناع الفرص" (Opportunity Makers)

هذا الملف يمثل دليلاً شاملاً لكود المشروع وهندسته المعمارية (Architecture) بحيث يسهل فهمه وتطويره مستقبلاً.

---

## 1. نظرة عامة على المشروع (Overview)
منصة "صناع الفرص" هي نظام متكامل لإدارة المتطوعين، اللجان، المهام، وتقييم الأداء (VPI). النظام ينقسم إلى:
- **واجهة أمامية (Frontend):** مبنية باستخدام React + TypeScript + Vite + TailwindCSS.
- **واجهة خلفية (Backend):** مبنية باستخدام Node.js + Express + Prisma + SQLite.
- **قاعدة البيانات (Database):** تستخدم SQLite مع Prisma ORM.

---

## 2. الواجهة الخلفية (Backend)
الباك إند مبني بطريقة معيارية (Modular) لتسهيل الصيانة.

### 2.1 مسارات الـ API (Routes)
جميع المسارات تبدأ بـ `/api/`، وتتضمن:
- `auth`: لتسجيل الدخول (Login)، إنشاء حساب (Signup)، وتغيير كلمة المرور.
- `users`: لجلب وتحديث بيانات المستخدم وملفه الشخصي.
- `admin`: مخصصة للأدمن فقط (قبول/رفض الأعضاء، التحليلات، إحصائيات اللجان).
- `tasks`: لإنشاء المهام (Admin)، تسليم المهام (User)، وتقييمها `gradeSubmission`.
- `evaluations`: لتسجيل التقييمات الشهرية وإرجاعها `getMyEvaluations`.
- `vpi`: المحرك الخاص بحساب مؤشر أداء المتطوع (Volunteer Performance Index).
- `scoring`: محرك النقاط (Gamification) الذي يوزع الشارات والمستويات.

### 2.2 المحركات الأساسية (Engines & Services)
- `scoringEngine.ts`: يحسب النقاط التراكمية بناءً على حضور الاجتماعات وإنجاز المهام.
- `vpiEngine.ts`: معادلة معقدة تحسب الـ VPI بناءً على (نقاط المهام GPA، التقييم الشهري، ونسبة إنجاز الساعات).
- `gamificationEngine.ts`: يوزع الشارات (Badges) ويرفع مستوى المستخدم (Beginner -> Intermediate -> Pro -> Leader) عند تجاوز نقاط معينة.

---

## 3. الواجهة الأمامية (Frontend)

### 3.1 الصفحات الأساسية (Pages)
- `HomePage.tsx`: الصفحة الرئيسية (Landing Page) بتصميم عصري (Animations).
- `AdminDashboard.tsx`: لوحة تحكم الإدارة الكاملة (فيها تبويبات: Pending Users, Tasks, Evaluations, AI Insights, Reports).
- `UserDashboard.tsx`: لوحة تحكم العضو المركزية. بتعرض الـ VPI، الـ Tasks الخاصة بيه، التقييمات، وإمكانية طلب مستندات (Documents).
- `CommitteeDashboard.tsx`: دايناميك. بتعرض لوحة تحكم مخصصة لكل لجنة (HR, PR, OR, Training, Social) بناءً على إعدادات الـ Config.
- `NotFoundPage.tsx`: صفحة الـ 404 للروابط الخاطئة.

### 3.2 إعدادات اللجان (Committee Config)
الموجود في `src/config/committeeConfig.ts`. ده بيتحكم في شكل كل لجنة:
- ألوان الـ Theme لكل لجنة مختلفة (مثلاً الأزرق للـ HR، البرتقالي للـ OR).
- بيحدد إيه الـ KPIs والأزرار السريعة المتاحة لكل لجنة.

### 3.3 حماية المسارات (Route Guards)
- يتم استخدام `ProtectedRoute` لضمان عدم دخول غير المسجلين.
- يتم استخدام `MustChangePwdGuard` لإجبار المستخدمين الجدد على تغيير الـ Temporary Password الخاص بهم (والذي يكون عادة `Sana3Admin2024!`) قبل استخدام النظام.

---

## 4. قاعدة البيانات (Prisma Schema)
أهم الجداول (Models) الموجودة في النظام:

- **`User`**: يحتوي على بيانات المتطوع، نقاطه الحالية (`points`, `score`, `level`) ومؤشرات الـ VPI التراكمية. يربط بين المتطوع ولجنته `Department`.
- **`Department`**: يمثل اللجان الخمس (HR, PR, OR, Training, Social).
- **`Task` & `TaskSubmission`**: الجدول الأول لتعريف المهمة بواسطة الأدمن، والثاني لتخزين ملف العضو مع التقييم (Grade: A, B, C, D, F).
- **`MonthlyEvaluation`**: التقييم الشهري المكون من 10 معايير أساسية.
- **`VpiRecord`**: سجل شهري يحتفظ بدرجة الـ VPI للمتطوع.
- **`OfficialDocument`**: يسجل أي شهادات (تطوع أو خبرة) يطلبها المستخدم وتم إصدارها.

---

## 5. رحلة المستخدم داخل النظام (User Flow)
1. **التسجيل (Signup):** المستخدم يسجل ويصبح في حالة `PENDING`.
2. **القبول (Approval):** الأدمن يدخل لوحة الإدارة ويوافق على حسابه، ويعين له كلمة مرور مبدئية `Sana3Admin2024!`.
3. **الدخول الأول (First Login):** النظام يجبر المستخدم على تغيير كلمة المرور.
4. **لوحة المستخدم:** المستخدم يدخل لوحته، أو لوحة لجنته، يرى المهام المطلوبة.
5. **تسليم المهام:** يرفع ملف المهمة.
6. **التقييم:** الأدمن يقيم المهمة (Grade)، ثم يقيم المتطوع شهرياً.
7. **الـ VPI والنقاط:** النظام يحسب الـ Score ويعطيه شارات (Badges) ويرفع درجته.

---

## 6. التحسينات الأخيرة التي تمت إضافتها
1. **الـ Navigation:** إمكانية تنقل الأدمن بسهولة من `User Dashboard` للـ `Admin Panel`، والعكس للمتطوع.
2. **ربط الـ Tasks:** إمكانية تقييم المهام من قِبل الأدمن بضغطة زر داخل شاشة العرض.
3. **ربط الـ Evaluations:** عرض التقييمات الشهرية بشكل رسومي مبسط في لوحة تحكم المستخدم بمجرد إصدارها.
4. **تنظيف الكود:** إزالة كافة تحذيرات وأخطاء `TypeScript` لضمان بناء (`build`) المشروع بنجاح.
